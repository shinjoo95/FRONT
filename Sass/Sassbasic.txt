1. Sass : CSS 대용 언어 
    - 조건문, 반복문, 변수, 함수 같은 프로그래밍스러운 문법이 존재함
        - 반복적인 부분을 쉽게 처리 가능 
        - 큰 프로젝트에서 특히 도움 됨 

    - Sass 파일명은 .scss
        - 웹브라우저는 .css만 읽을 수 있음 / .scss는 읽을 수 없음
            - scss ---> css 변환해야 html에 사용 가능 
                .map 파일의 용도 : 크롬 개발자도구 디버깅용 
                    - .map 파일이 있으면 크롬에서 css가 아니라 scss 파일로 분석해줌 

    1. 어려운 단어 기억해야할 땐 변수 문법을 씀
        - $변수명 : 저장해서 쓸 값;
    2. Sass 에선 사칙연산 바로 가능 ()괄호 쓰는 버릇 들이기 
        - CSS 기본 문법에서도 변수 문법이 존재 
            :root{
                --main-color : red;
            }
            .background{
                background-color : var(--main-color);
                font-size : calc(40px - 20px);
            }
    3. nesting 문법
        - 관련있는 class들 묶을 때 좋음
        - 2개 이상은 가능한 X
    4. @extend 문법
        - 중복을 제거하기 위해 뼈와 살을 나누어 짬
        - 중복된 스타일이 많으면 임시 클래스를 만들어 복사해 사용 
        - class 전체를 복사해주는 용도
        - %임시클래스는 단독으로 컴파일되지 않음 / 다른곳에 종속된 클래스를 보관해두고 싶으면 사용 
    5. mixin 문법
        - 긴 코드를 짧은 단어로 축약할 때 사용
        - $파라미터 긴 코드를 가변적으로 만들 때 씀
    6. import 문법
        - 다른 파일에 있는 내용을 가져오고 싶을 때 사용
        - @use '파일경로';
        - 참조 파일은 굳이 컴파일 해줄 필요가 없기 때문에 _파일명.scss로 하면 css파일로 컴파일을 안해줌
        - 참조 파일의 변수도 사용 가능 
            - reset.$메인컬러 
            - 다른 파일의 @mixin 쓰려면 @include 파일명.mixin이름
    7. function, if, for 은 거의 사용을 하지 않음. 
        
        
2 .scss / .sass 파일의 차이 
    - .sass 파일은 괄호를 안써도 됨
    - 예제코드 복붙은 모두 scss 괄호가 있음
